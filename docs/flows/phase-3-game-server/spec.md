# Phase 3: 게임 서버 코어

## 목표
WebSocket 룸 관리, 60초 턴 스케줄러, 전투 판정, 승리 조건, Redis 상태 저장을 구현한다.

## 확정된 게임 수치

### 맵
| 항목 | 값 |
|------|-----|
| 맵 크기 | 800×800px (소형) |
| 벽/장애물 | 심플 기둥/잔해 배치 (소수, 개방감 유지) |

### 기본 전투 수치
| 항목 | 값 |
|------|-----|
| 캐릭터 HP | 100 |
| 근접 공격 사거리 | 80px |
| 근접 데미지 | 15/턴 |
| 원거리 사거리 | 300px |
| 원거리 데미지 | 10/턴 |
| 이동 속도 | 120px/턴 |

## 작업 목록

| # | 작업 | 산출물 |
|---|---|---|
| 3-1 | WebSocket 서버 + 룸 관리 (매치별) | ws 연결, 입장/퇴장 처리 |
| 3-2 | 60초 턴 스케줄러 | 타이머 → 캐릭터 순회 → 상태 업데이트 |
| 3-3 | 충돌 처리 (이동/전투 판정, 근접·원거리) | 전투 로직 모듈 |
| 3-4 | 사망 처리 + 최후 1인 게임 종료 | 승리 조건 판정 |
| 3-5 | Redis 상태 저장/복원 | 턴 간 게임 상태 persistence |

## 완료 기준
- [ ] 복수 클라이언트 WebSocket 연결 + 룸 입장/퇴장 처리
- [ ] 60초 주기로 모든 캐릭터 행동 순차 처리 후 상태 브로드캐스트
- [ ] 전투 판정: 근접(80px) / 원거리(300px) 사거리 기반 데미지 계산 + HP 0 → 사망 처리
- [ ] 마지막 생존자 발생 시 게임 종료 이벤트 전송
- [ ] 서버 재시작 후 Redis에서 게임 상태 복원 가능
