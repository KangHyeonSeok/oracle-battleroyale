# Phase 4: 신탁 시스템

## 목표
신탁 채팅 수신 → Gemini OracleIntent 추출 → credulity 판정 → override_queue 삽입 → 포인트 관리 파이프라인을 구현한다.

## 신탁 유형

| 유형 | 대상 | 예시 |
|---|---|---|
| 강화 | 본인 캐릭터 | 지금 도망쳐 |
| 방해 | 타인 캐릭터 | 저 녀석은 겁쟁이야 |
| 유혹 | 타인 캐릭터 | 저쪽에 보물이 있어 |

방해/유혹 성공 여부는 대상 캐릭터의 credulity 파라미터로 판정.

## 작업 목록

| # | 작업 | 산출물 |
|---|---|---|
| 4-1 | 신탁 채팅 수신 → Gemini OracleIntent 추출 | 자연어 → JSON 파이프라인 |
| 4-2 | credulity 성공 판정 로직 | 확률 계산 모듈 |
| 4-3 | override_queue 삽입 → 다음 턴 반영 | 큐 관리 |
| 4-4 | 분당 1회 쿨다운 + 성좌 포인트 차감(-5pt) | 서버 타임스탬프 기준 |
| 4-5 | 성좌 포인트 적립 (생존 +1, 승리 +5, 역전 +10) | 이벤트 기반 적립 |

## 성좌 포인트 규칙

| 이벤트 | 포인트 |
|---|---|
| 신탁 1회 전송 | -5pt |
| 본인 캐릭터 생존 중 | 턴당 +1 |
| 전투 승리 | +5 |
| 신탁으로 극적 역전 | +10 |
| 관전 중 채팅 반응 | +1 |

## 완료 기준
- [ ] 신탁 메시지 → Gemini 분류 → 유형(강화/방해/유혹) + 대상 추출
- [ ] credulity 판정: 성공 시 override_queue에 삽입, 다음 턴 AI 행동에 반영
- [ ] 쿨다운: 동일 플레이어 분당 1회 초과 시 거부 응답
- [ ] 포인트 부족 시 신탁 거부, 포인트 차감 정확성 검증
